# Crawler Scrapy + Selenium

## Описание
Этот проект – краулер, построенный на **Scrapy** и **Selenium**.  
Паук `wb_spider` выполняет следующие шаги:

1. **Генерирует ссылку каталога товаров** по заданному поисковому запросу.  
2. **Скроллит страницу с каталогом**, подгружая все товары.  
3. **Переходит на карточку каждого товара**, парсит нужные данные и сохраняет их.  

---

## Инструкция по запуску

### 1. Установка зависимостей
```bash
# Перейти в корень проекта
cd <папка проекта>

# Создать виртуальное окружение (пример для venv)
python -m venv .venv

# Активировать окружение
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

# Установить зависимости
pip install -r requirements.txt
```

### 2. Запуск краулера
```bash
# (если вы вышли из окружения – активируйте его снова)
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

# Запуск паука
scrapy crawl wb_spider
```

> **Примечание:** При первом запуске потребуется установить драйвер Chrome, совместимый с вашей версией браузера. Если используется `webdriver-manager`, он скачает нужную версию автоматически.

## Собранные данные

Для каждого товара краулер сохраняет следующие поля:

| Поле | Описание |
|------|----------|
| **Ссылка на товар** | Прямая ссылка на карточку товара. |
| **Артикул** | Артикул товара. |
| **Название селлера** | Название продавца (селлера). |
| **Ссылка на селлера** | Ссылка на профиль продавца. |
| **Рейтинг** | Средний рейтинг товара (0‑5). |
| **Количество отзывов** | Количество отзывов/оценок. |
| **Цена** | Текущая цена (в валюте сайта). |
| **Размер** | Все размеры (даже если нет в наличии), разделённые запятой. |
| **Остатки** | Остатки товара для каждого размера, разделённые запятой. |
| **Название** | Наименование товара. |
| **Описание** | Текстовое описание товара. |
| **Характеристики** | Характеристики товара; каждая характеристика разделяется символом новой строки (`\n`). |
| **Ссылки на изображения** | URL‑ы изображений товара, разделённые запятой. |

Результат сохраняется в формате, указанном в `settings.py` (`output/items.json`).

## Особенности реализации

- **Многопоточность** – использует Twisted‑reactor Scrapy с настроенным `CONCURRENT_REQUESTS_PER_DOMAIN = 2` (два потока).  
- **Selenium WebDriver** – для каждого HTTP‑запроса создаётся новый экземпляр `ChromeDriver`. После сбора всех данных драйвер закрывается, что изолирует сессии и уменьшает риск блокировки.  
- **Ротация User‑Agent** – middleware подменяет заголовок `User-Agent` на случайный из списка, чтобы снизить вероятность детекции бота.  
- **Гибридный парсинг** – данные извлекаются как обычным поиском элементов (`response.css` / `response.xpath`), так и перехватом XHR/Fetch‑запросов через собственный лог драйвера.  
- **Сохранение HTML при ошибке** – в случае критического исключения (например, таймаут загрузки страницы) текущий HTML‑код сохраняется в файл `errors/fatal_error_snapshot.html` для последующего анализа.  
- **Настраиваемые параметры** – параметры паука задаются в `settings.py`.
